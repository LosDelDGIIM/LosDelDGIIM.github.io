1. Si comparamos el algoritmo Descenso Iterativo \\(A^\ast\\) con la Búsqueda Primero el Mejor Recursiva:
    - ( ) no se pueden comparar en términos de memoria
    - ( ) el segundo requiere almacenar menos nodos en memoria, pero podría necesitar regenerar más nodos
    - (x) el primero requiere almacenar menos nodos en memoria, pero podría necesitar regenerar más nodos

<details>
  <summary>Respuesta</summary>
  <p>El Descenso Iterativo \\(A^\ast\\) (ID\\(A^\ast\\)) trabaja con iteraciones de profundidad limitada por coste f, manteniendo en memoria únicamente el camino actual, lo que supone un consumo muy reducido. La BPMR (RBFS) mantiene en cada nivel el mejor nodo alternativo conocido. Por tanto, ID\\(A^\ast\\) (el primero) almacena menos nodos, pero al reiniciar las iteraciones puede regenerar muchos más nodos que la BPMR.</p>
</details>

    - ( ) tratar con las precondiciones no explícitamente descritas en la acción cuando ésta falla
    - (x) deducir de forma eficiente aquello que no cambia cuando se aplica una acción

<details>
  <summary>Respuesta</summary>
  <p>El problema del marco (frame problem) trata sobre cómo representar y razonar eficientemente sobre lo que NO cambia al aplicar una acción. En lógica clásica es necesario especificar explícitamente todo lo que permanece igual, lo que genera una cantidad enorme de axiomas. Sistemas como STRIPS abordan esto mediante la hipótesis del mundo cerrado.</p>
</details>

    - ( ) mediante el uso de un valor constante mayor que cualquier valor de coste posible de una solución
    - ( ) no contempla nodos irresolubles ya que estos no aparecen en problemas descomponibles

<details>
  <summary>Respuesta</summary>
  <p>En Y/O*, cuando se detecta que un nodo no tiene solución (es irresoluble), esta información se propaga hacia arriba en el grafo Y/O. Un nodo OR es irresoluble si todos sus hijos son irresolubles; un nodo AND es irresoluble si alguno de sus hijos lo es. Este mecanismo de propagación permite podar ramas sin solución de forma eficiente.</p>
</details>

11. La Búsqueda Primero el Mejor Recursiva (BPMR) requiere de un mecanismo de propagación basado en el:
    - ( ) promedio
    - (x) máximo
    - ( ) mínimo

<details>
  <summary>Respuesta</summary>
  <p>En la BPMR (RBFS), cuando se abandona un camino para explorar otro más prometedor, el valor del nodo abandonado se actualiza con el mínimo de sus hijos. Sin embargo, el mecanismo de propagación hacia arriba al retornar de la recursión se basa en el máximo: el padre refleja el mejor coste alcanzable desde él, que es el mínimo de sus hijos, y ese valor se propaga hacia arriba como el máximo entre las alternativas disponibles.</p>
</details>

12. Si \\(h\\) es admisible y decidimos usar como función heurística para un problema \\(f(n) = g(n) + w\cdot h(n)\\) en un algoritmo \\(A^\ast\\), entonces se obtiene la solución óptima para los siguientes valores del parámetro w:
    - ( ) w mayor que 1
    - (x) w menor o igual que 1
    - ( ) w perteneciente al intervalo cerrado [0,1]
    - ( ) para cualquier valor del parámetro w

<details>
  <summary>Respuesta</summary>
  <p>Con f(n) = g(n) + w·h(n), si w ≤ 1 entonces w·h(n) ≤ h(n) ≤ \\(h^\ast\\)(n), de modo que la función sigue siendo admisible y \\(A^\ast\\) garantiza la solución óptima. Si w > 1 se obtiene \\(A^\ast\\) ponderado, que puede ser más rápido pero no garantiza la optimalidad. El intervalo [0,1] es un subconjunto correcto pero más restrictivo de lo necesario, ya que la condición real es simplemente w ≤ 1.</p>
</details>

13. En el modelo de regla tipo STRIPS la lista de supresión es:
    - ( ) una lista de predicados con variables cuantificadas universalmente que se usa para construir el estado en el que se ha aplicado la regla
    - ( ) una lista de predicados con variables cuantificadas existencialmente que se usa para construir el estado en el que se ha aplicado la regla
    - ( ) una fórmula bien formada de la lógica para determinar cuando se puede aplicar una regla
    - (x) una lista de predicados con variables libres que se usa para construir el estado en el que se ha aplicado la regla

<details>
  <summary>Respuesta</summary>
  <p>En STRIPS, una regla tiene tres componentes: precondición (fórmula lógica), lista de adición y lista de supresión (delete list). La lista de supresión contiene predicados con variables libres que se instancian al aplicar la regla, y que se eliminan del estado resultante. La precondición es la única parte que constituye una fórmula lógica; las listas de adición y supresión son simplemente listas de predicados con variables libres.</p>
</details>

27. Se produce una amenaza a un vínculo causal cuando:
    - ( ) hay una acción que elimina el literal protegido por el vínculo causal y dicha acción está ordenada antes de la primera acción del vínculo
    - ( ) hay una acción que añade el literal protegido por el vínculo causal y dicha acción podría ordenarse entre las dos acciones del vínculo

    - (x) hay una acción que elimina el literal protegido por el vínculo causal y dicha acción podría ordenarse entre las dos acciones del vínculo

<details>
  <summary>Respuesta</summary>
  <p>En planificación de orden parcial, un vínculo causal A →p B significa que A establece el predicado p necesario para B. Una amenaza ocurre cuando existe una acción C que elimina p y podría ordenarse entre A y B, interfiriendo con el vínculo. La solución es añadir restricciones de orden para resolver la amenaza: ordenar C antes de A o después de B.</p>
</details>

28. La hipótesis del mundo cerrado:
    - (x) permite manejar un tipo de incertidumbre muy limitada

    - ( ) permite manejar la incertidumbre usando probabilidades

<details>
  <summary>Respuesta</summary>
  <p>La hipótesis del mundo cerrado (CWA) asume que todo lo que no está demostrado como verdadero es falso. Esto permite manejar una forma muy limitada de incertidumbre: la ausencia de información se interpreta como falsedad. No usa disyunciones ni probabilidades, pero sí representa una gestión (muy restrictiva) de la falta de información.</p>

29. El problema de la cualificación en planificación está relacionado con:
    - ( ) los efectos de las acciones
    - (x) las precondiciones de las acciones
    - ( ) la eficiencia en el razonamiento sobre las acciones

<details>
  <summary>Respuesta</summary>
  <p>El problema de la cualificación trata sobre las precondiciones de las acciones: es imposible enumerar exhaustivamente todas las condiciones que deben cumplirse para que una acción se ejecute con éxito. 30. La diferencia entre usar el algoritmo \\(A^\ast\\) sobre grafos de visibilidad frente a cuadrículas en problemas de planificación de caminos es que:

    - (x) para los grafos de visibilidad \\(A^\ast\\) requiere más complejidad, aunque podría obtener una mejor solución
    - ( ) para los grafos de visibilidad \\(A^\ast\\) requiere más complejidad, aunque podría obtener una peor solución
    - ( ) para los grafos de visibilidad \\(A^\ast\\) requiere menos complejidad y podría obtener una peor solución

<details>
  <summary>Respuesta</summary>
  <p>Los grafos de visibilidad son más costosos de construir (O(n² log n)) y de explorar con \\(A^\ast\\) (mayor número de nodos y aristas) que una simple cuadrícula. Sin embargo, permiten encontrar caminos que siguen los contornos de los obstáculos de forma óptima, obteniendo soluciones más cortas (mejor solución) que las rutas discretizadas en cuadrícula, que están limitadas por la resolución de la malla.</p>
